## Phase 1 — Security hardening (do before wider distribution)

### 1.1 Verify checksums on ALL binary downloads — **HIGH, M**
Only the TSO assets download passes a checksum (`TsoInstaller.cs:59`). These do NOT:
- Launcher self-update: `SelfUpdateService.cs:60` — `new DownloadService(assetUrl, tmpZip)` (no hash)
- Game client: `FsoInstaller.cs:50`
- Game patches: `GameUpdateService.cs:77`
- Mono: `MonoInstaller.cs:46`, SDL: `SdlInstaller.cs:46`, RMS: `RmsInstaller.cs:62`

A MITM (hostile Wi-Fi, DNS hijack, compromised mirror) can swap in a trojaned binary that we
then execute. Fix: publish a SHA-256 per asset in the release (e.g. a `checksums.txt` asset
generated by CI, or use the GitHub API `digest` field where available), fetch it over HTTPS,
and pass it to `DownloadService`. Extend `DownloadService` to support SHA-256 alongside MD5.
